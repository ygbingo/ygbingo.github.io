> OpenClaw 3.8 (2026.3.8) 带来了大量新功能、性能优化和安全性提升。本文将为你详细解读这个版本的重大更新，帮助你快速上手新特性。

## 🚀 核心新功能

### 1. 本地备份系统 (`openclaw backup`)

**全新 CLI 备份命令**，让你的数据更安全：

```bash
# 创建完整备份
openclaw backup create

# 仅备份配置
openclaw backup create --only-config

# 排除工作空间
openclaw backup create --no-include-workspace

# 验证备份完整性
openclaw backup verify
```

**特性亮点**：
- 自动归档命名（支持日期排序）
- 清单/负载验证
- 在破坏性操作前提供备份指导
- 配置专属备份模式

---

### 2. Talk 模式静音超时配置

新增 `talk.silenceTimeoutMs` 配置，让语音对话更智能：

```yaml
talk:
  silenceTimeoutMs: 2000  # 2秒静音后自动发送
```

**使用场景**：
- 控制语音输入的自动提交时机
- 各平台保持默认暂停窗口
- 避免误触发送

---

### 3. TUI 智能代理推断

TUI 现在能**自动识别当前工作空间**的活跃代理：

- 在配置好的代理工作空间内启动 TUI 时，自动推断活跃代理
- 保留显式 `agent:` 会话目标
- 减少手动切换代理的步骤

---

### 4. Brave 搜索 LLM 上下文模式

**可选的 `llm-context` 模式**，让搜索结果更智能：

```yaml
tools:
  web:
    search:
      brave:
        mode: "llm-context"  # 提取结构化片段和来源元数据
```

**优势**：
- 调用 Brave 的 LLM Context 端点
- 返回提取的 grounding snippets
- 带来源元数据，便于引用

---

### 5. ACP 溯源元数据

新增 **ACP 入口溯源支持**：

```bash
# 关闭溯源
openclaw acp --provenance off

# 仅元数据
openclaw acp --provenance meta

# 元数据 + 可见收据
openclaw acp --provenance meta+receipt
```

**功能**：
- 保留 ACP 来源上下文
- 会话追踪 ID
- 可见收据注入

---

### 6. macOS 远程网关令牌

**远程模式支持**：
- 添加远程网关令牌字段
- 保留现有的非明文 `gateway.remote.token` 配置值
- 加载令牌形状无法直接使用时发出警告

---

## 🛠️ 重要改进

### 搜索引擎提供商排序优化

搜索引擎提供商现在**按字母顺序排列**：
- 保持提供商列表中立性
- 多键自动检测现在优先选择 Grok 而非 Kimi
- 配置选择器中的排序更一致

---

### 版本输出增强

`openclaw --version` 现在包含**短 Git 提交哈希**：

```bash
$ openclaw --version
openclaw 2026.3.8 (abc1234)
```

便于精确追踪版本和调试。

---

### 浏览器代理绑定主机配置

新增 `browser.relayBindHost` 配置：

```yaml
browser:
  relayBindHost: "0.0.0.0"  # 支持 WSL2 和跨命名空间设置
```

**解决场景**：
- WSL2 与 Windows Chrome 通信
- 跨命名空间设置
- 保持默认仅环回行为

---

### 主题自动检测

TUI 现在**自动检测终端背景色**：

- 通过 `COLORFGBG` 检测浅色终端背景
- 选择符合 WCAG AA 标准的浅色/深色调色板
- 支持 `OPENCLAW_THEME=light|dark` 手动覆盖

---

## 🔒 安全性提升

### 网关认证模式强制要求

⚠️ **破坏性变更**：当同时配置 `gateway.auth.token` 和 `gateway.auth.password` 时，必须显式设置 `gateway.auth.mode`：

```yaml
gateway:
  auth:
    mode: "token"  # 或 "password"
    token: "your-token"
    # password: "your-password"  # 二选一
```

**升级前请设置**，否则会导致启动/配对/TUI 失败。

---

### 私有网络重定向拦截

浏览器工具现在**阻止私有网络中间重定向跳转**：

- 严格浏览器导航流程中拦截
- 远程标签页打开路径无法检查重定向链时失败关闭
- 防止 SSRF 攻击

---

### 系统运行脚本绑定

批准的 `bun` 和 `deno run` 脚本操作数现在**绑定到磁盘文件快照**：

- 批准后脚本重写被拒绝
- 执行前检查文件完整性
- 防止脚本篡改攻击

---

### 技能下载安装加固

下载安装现在**固定验证的每技能工具根目录**：

- 写入前固定词法工具路径
- 防止重定向下载写入到非预期目录
- 感谢 @tdjackey 的贡献

---

## 🐛 关键修复

### 消息平台修复

| 平台 | 修复内容 |
|------|----------|
| **Telegram** | DM 路由去重、话题线程绑定、部分流式修复 |
| **Slack** | `app_mention` 竞态去重、反应线程上下文路由 |
| **Matrix** | DM 路由安全回退、房间绑定保留 |
| **Feishu** | 流卡片交付优化、群组提及检测修复 |
| **LINE** | `requireMention` 群组门控对齐 |
| **Mattermost** | 模型选择器交互修复、线程回复保持 |

---

### 浏览器工具修复

1. **CDP WebSocket 规范化**
   - 将通配符 `ws://0.0.0.0` 和 `ws://[::]` 调试器 URL 重写回外部 CDP 主机/端口
   - 修复 Browserless 风格容器端点

2. **标签页瞬断重连**
   - 短暂中继断开后等待先前附加的 Chrome 标签页重新出现
   - 减少嘈杂的重连闪烁

3. **会话清理**
   - 会话重置/删除时关闭会话范围的浏览器标签页
   - 防止孤立标签页和无界浏览器内存增长

---

### 网关稳定性修复

- **更新重启**：重新启用禁用的 LaunchAgent 服务，使 `openclaw update` 能从禁用状态恢复
- **重启超时恢复**：重启触发关闭超时时非零退出，使 launchd/systemd 能重启网关
- **配置重启保护**：启动/重启前验证配置，防止无效配置重启循环
- **启动守卫**：将 `XPC_SERVICE_NAME` 视为 launchd 监管提示

---

### 内存与上下文修复

- **混合搜索**：保留负 FTS5 BM25 相关性排序
- **QMD 搜索结果解码**：接受仅包含文件 URI 的命中
- **QMD 集合名称冲突恢复**：检测冲突集合并重试添加
- **上下文修剪**：防护助手思考/文本字符估算免受畸形块影响

---

### 模型与代理修复

- **Kimi Coding**：规范化 `anthropic-messages` 工具负载到 OpenAI 风格
- **xAI 工具调用**：解码 HTML 实体编码的参数值
- **Ollama**：远程提供者认证回退、自定义头部转发、压缩和摘要处理
- **OpenAI Codex**：OAuth/登录一致性、令牌刷新加固
- **Venice**：模型限制对齐、工具连线禁用

## 📝 升级建议

### 升级前检查清单

1. **备份配置**
   ```bash
   openclaw backup create --only-config
   ```

2. **检查网关认证配置**
   - 如果同时有 `token` 和 `password`，设置 `gateway.auth.mode`

3. **更新后验证**
   ```bash
   openclaw status
   openclaw doctor
   ```

---

### 推荐新配置

```yaml
# Talk 模式静音超时
talk:
  silenceTimeoutMs: 2000

# Brave 搜索 LLM 上下文模式
tools:
  web:
    search:
      brave:
        mode: "llm-context"

# 浏览器中继绑定（WSL2 用户）
browser:
  relayBindHost: "0.0.0.0"

# 网关认证模式（必须）
gateway:
  auth:
    mode: "token"  # 或 "password"
```
